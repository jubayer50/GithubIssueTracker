1. What is the difference between var, let, and const?
   Ans:
   var, let, and const are used to declare variables in JavaScript. Though they are used to do same things, they have some differences from each other. var was the only variable declaration key before updating ES6 in JavaScript. After ES6 Updating in JavaScript introduced let and const variable declarations. var is a functional scoped, but let and const are not; they are block-scoped. We can re-declare with var, but we can not with let and const. Also, we can reassign with var and let, but not with const. The crucial difference is that all of them are hoisted but not initialized without var.

2. What is the spread operator (...)?
   Ans:
   Spread Operator (...) is used to expand elements of an array and object in JavaScript. We can use the spread operator when we copy objects and arrays into a variable so that we can update and change them independently. We also use a function argument.

3. What is the difference between map(), filter(), and forEach()?
   Ans:
   These are all array methods. All are working like a loop. Map method run loop over an array, and it works with every individual element in the array and returns them. Filter methods can separate conditional ways in the array elements, then return them. ForEach methods also work as a map method, but it does not return.

4. What is an arrow function?
   Ans:
   An arrow function works like a function. It is a short syntax of a function that was introduced ES6. It helps to write a simple and easy function. If we write a function in one line by using an arrow function, we do not need to write return in the function (auto return). We can write a function with parameters and without parameters using an arrow function. It looks clean and easy to read.

5. What are template literals?
   Ans:
   Template literal is a way of writing a string in JavaScript. To write a string using template literal need backticks. When we use template literal we can use variables dynamically, and it is one of the most powerful reason of using template literals. By using template literals, we can write multi line string, which is not possible using double and single quotation marks. It also gives us a clean look and makes our code easy to read.
